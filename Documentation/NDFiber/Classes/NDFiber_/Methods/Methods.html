continue

yields the current fiber to the receiver

Author: Nathan Day

- (void)continue;
Discussion

This is the method in which the switching between fibers occurs, +[NDFiber yieldToFiber:] sends this message to its argument. If the receiver is finished then the exception NDFiberSucceedToFinishedFiberException is thrown

detachNewFiberSelector:toTarget:withObject:

Detaches a new fiber and uses the specified selector as the fiber entry point.

Author: Nathan Day

+ (NDFiber *)detachNewFiberSelector:(SEL)selector toTarget:(id)target 
        withObject:(id)argument;
Parameters
selector
The selector for the message to send to the target. This selector must take only one argument and must not have a return value.
target
The object that will receive the message aSelector on the new fiber.
argument
The single argument passed to the target. May be nil.
Discussion

For non garbage-collected applications, the method selector can set up an autorelease pool for the newly detached fiber and freeing that pool before it exits, but unlike threads this is not required. Also the fiber returned is autoreleased and so you need to retain it like you retain any other object. The objects aTarget and anArgument are retained during the execution of the detached fiber, then released. The detached fiber is exited as soon as aTarget has completed executing the aSelector method.

fiberDictionary

Returns the fiber object's dictionary.

Author: Nathan Day

- (NSMutableDictionary *)fiberDictionary;
Return Value

The fiber object's dictionary.

Discussion

You can use the returned dictionary to store fiber-specific data. The fiber dictionary is not used during any manipulations of the NDFiber object—it is simply a place where you can store any interesting data. You may define your own keys for the dictionary.

setStackSize:

Sets the stack size of the receiver. This value will be rounded up to the nearest multiple of 4KB.

Author: Nathan Day

- (void)setStackSize:(NSUInteger)size;
Parameters
size
The stack size for the receiver.
Discussion

You must call this method before starting your fiber. Setting the stack size after the fiber has started changes the attribute size (which is reflected by the stackSize method), but it does not affect the actual number of pages set aside for the fiber.
